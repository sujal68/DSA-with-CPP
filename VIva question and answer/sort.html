Array kya hai?
"Array ek fixed-size, linear data structure hota hai jo same type ke elements ko continuous memory me store karta hai."
Key Points:
  • Array ka size fixed hota hai
  • Saare elements same data type ke hote hain
  • Memory contiguous hoti hai (side-by-side blocks)
  • Access index se direct hota hai (0-based indexing)
Example: int a[5] ek array hai, jisme 5 integers store hote hain, aur a[0] first element hota hai.
Sorting arrays par isliye fast hoti hai kyunki array contiguous memory deta hai, elements ka direct access mil jata hai, aur compare/swap operations easy hote hain.

Compiler

"Pura code ek saath translate karta hai aur executable file banata hai."

  • Fast execution
  • Error detection pehle
  • Machine code generate karta hai

Interpreter

"Code ko line-by-line translate karke turant execute karta hai."

  • Slow execution
  • Error detection line-by-line
  • No separate executable file
C aur C++ me difference

C Language

  • Procedure-Oriented Language
  • Top-Down approach
  • No OOP concepts
  • Less secure
  • System programming, OS

C++ Language

  • Object-Oriented Language
  • Bottom-Up approach
  • Full OOP support
  • More secure (encapsulation)
  • Game dev, OOP applications
C procedure-oriented hai, C++ object-oriented hai; C me classes nahi hoti, C++ me OOP concepts hote hain.

üîç Searching & Algorithms
Searching kya hai?
"Searching ka matlab kisi data structure me kisi specific element ko dhoondhna."
Types of Searching:
  • Linear Search: Element ko ek-ek karke check karke dhoondte hain
  • Binary Search: Sorted data me mid ke basis par fast searching hoti hai
BST (Binary Search Tree) kya hai?
"BST ek tree hota hai jisme left side ke nodes chhote hote hain aur right side ke nodes bade hote hain."
BST me searching root se start hoti hai; element chhota ho to left jate hain, bada ho to right jate hain ‚Äî jab tak element mil na jaye.
Time Complexity:
  • Best / Avg: O(log n)
  • Worst: O(n) (jab tree skewed ho)
BFS (Breadth-First Search) kya hai?
"BFS ek graph/tree traversal technique hai jisme hum nodes ko level-by-level, breadth-wise visit karte hain."
Kaise kaam karta hai:
  • BFS queue use karta hai
  • Pehle root ko queue me daalte hain
  • Phir uske saare neighbours ko visit karte hain
  • Phir unke neighbours ‚Äî level by level
Where BFS is used:
  • Shortest path find karne ke liye
  • Social network suggestions
  • Maps / GPS route
  • Level-order traversal in trees
Time Complexity: O(V + E) (vertices + edges)
DFS (Depth-First Search) kya hai?
"DFS ek traversal technique hai jisme hum ek path ko maximum depth tak explore karte hain, phir backtrack karke next path follow karte hain."
Kaise kaam karta hai:
  • DFS stack use karta hai (ya recursion)
  • Node visit karo
  • Uske deepest child tak jao
  • Jab aur aage nahi ja sakte, backtrack karo
  • Next path follow karo
Where DFS is used:
  • Cycle detection
  • Backtracking (Sudoku, Maze)
  • Topological sort
  • Connected components find karne me

Ambiguity Error kab aati hai?
"Ambiguity error tab aati hai jab compiler ko ye samajh na aaye ki kaun sa function ya kaun sa meaning choose karna hai ‚Äî yani multiple possible matches ho jaate hain."
Common Cases:
  • Function overloading me: Jab do functions bilkul similar hon
  • Multiple inheritance me: Jab do parent classes me same named function ho
Example - Function Overloading: void fun(int); void fun(float); fun(5); // int bhi ho sakta hai, float bhi ‚Äî ambiguity
Example - Multiple Inheritance: class A { void show(); } class B { void show(); } class C : public A, public B {}; C obj; obj.show(); // A ka? B ka? ‚Üí ambiguity
Virtual Method se Ambiguity kaise solve karein?
"Jab multiple inheritance me same function do parent classes me ho, to unhe virtual bana dene se compiler ko pata chal jata hai ki actual function ka base version ek hi hai ‚Äî aur ambiguity khatam ho jaati hai."
Virtual method ek shared function table banata hai, jisse multiple inheritance me function call clear hota hai aur ambiguity error nahi aati.
#include <iostream> using namespace std; class A { public: virtual void show() { cout << "A ki show()" << endl; } }; class B : public A { public: void show() override { cout << "B ki show()" << endl; } }; class C : public B {}; int main() { C obj; obj.show(); // No ambiguity return 0; }
